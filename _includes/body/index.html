{% comment %} Project override of hydejack/_includes/body/index.html. Adds per-viewport cover control on top of Hydejack's `cover` flag: desktop_cover: true|false # show the cover hero on desktop mobile_cover: true|false # show the cover hero on mobile When EITHER flag is true, we set Hydejack's `effective_cover` to true so the theme renders the standard cover layout (sidebar BEFORE main, .cover class on hy-drawer, opened attribute) — i.e. exactly Hydejack's `cover: true` behavior. CSS in _includes/my-head.html then reverts the .cover styling on the viewport whose flag is false (e.g. on desktop when desktop_cover is false, hy-drawer.cover is styled like the non-cover side panel — same visual result as `cover: false`). When both flags are false, falls back to page.cover so existing pages keep working unchanged. The rest of this file mirrors the theme version verbatim. {% endcomment %} {% include_cached pro/dark-mode-fix.html %} {% if page.desktop_cover or page.mobile_cover %} {% assign effective_cover = true %} {% else %} {% assign effective_cover = page.cover %} {% endif %} {% assign assets_url = "/assets/" | relative_url %} {% capture sidebar %}{% include_cached body/sidebar.html cover=effective_cover invert=page.invert_sidebar theme_color=page.theme_color image=image color=color %}{% endcapture %} {% if effective_cover %}{{ sidebar }}{% endif %} {% include_cached body/menu.html %} {% include body/main.html %} {% unless effective_cover %}{{ sidebar }}{% endunless %} {% unless page.redirect %} {% include_cached body/scripts.html %} {% include my-body.html %} {% endunless %} {% if page.desktop_cover or page.mobile_cover %} {% assign _bp = page.cover_breakpoint | default: site.hydejack.break_point_3 | default: 64 %} {% endif %} {% include_cached templates/index.html %}
