The schema is documented in the file's header comment. Author handling: The `authors` string is a plain comma-separated list — no "and" before the last name and no … around the homepage owner. Both are added automatically here. • Owner detection. Every `name` listed under `site.data.authors` is treated as the homepage owner (currently "Donny Y. Chen" and "Yuedong Chen" via author1/author2). When a token's clean name matches one of those, it is wrapped in … (preserving any annotation suffix like * or †) and not linked. • HTML pass-through. Tokens that already contain a tag — `…` on legacy entries, an intentional `et al.` elision — are emitted verbatim so we never wrap a tag inside an anchor. • Co-author links. Anything else is wrapped in ``. The clean name (with * and † stripped) is looked up in `_data/coauthors.yml` and that URL is used if present; otherwise we fall back to a Google search for the name. • "and" conjunction. When the entry has more than one author the last token is automatically prefixed with "and ".
